I have a task related to machine learning:
Build a universal Medical Model-to-Skill wrapping pipeline and an LLM-powered Medical Agent that can discover, select, and compose Medical Skills for complex clinical workflows, targeting MICCAI 2026.

Core Idea: Design a generic wrapping framework that can automatically convert any medical AI model (segmentation, classification, detection, registration, report generation, VQA, etc.) into a standardized Medical Skill using the Model Context Protocol (MCP) as the interoperability layer. The pipeline takes a medical model (with its code, weights, and API documentation) as input and produces a self-describing MCP-compatible Medical Skill as output — complete with structured input/output schemas, medical metadata (modality, anatomy, task type), safety constraints, and validation checks.

Key Components:
(1) Med-SkillGen: A universal wrapping pipeline (inspired by TxAgent's ToolGen) that auto-generates MCP-compatible Medical Skills from existing medical AI models. This includes: (a) automatic API analysis and schema extraction from model code/documentation, (b) medical metadata annotation (imaging modality, anatomical region, clinical task type, safety level), (c) standardized I/O interface generation following MCP tool specification, and (d) validation test generation to ensure skill correctness.
(2) Med-SkillLib: A curated Medical Skills library built by applying Med-SkillGen to existing open-source medical models (MedSAM, MedSAM2, CheXagent, etc.) and existing Medical MCP servers (DICOM-MCP, healthcare-mcp-public, medical-calculator-MCP, etc.). The library organizes skills hierarchically by modality, anatomy, and task type.
(3) Med-Agent: An LLM-powered agent that uses active skill discovery (inspired by MCP-Zero) to dynamically retrieve and compose Medical Skills from Med-SkillLib for multi-step clinical workflows. The agent performs task planning, skill selection via hierarchical semantic routing, execution graph construction, and result integration.

The key innovation bridges two worlds: (a) general-purpose agent skill frameworks (ToolLLM, AgentSkillOS, SkillRL, MCP-Zero) that lack medical domain expertise, and (b) isolated medical AI tools that lack interoperability and composability. By using MCP as the universal protocol, any medical model can be 'plugged in' as a skill, enabling the agent to solve complex clinical scenarios through dynamic skill composition.

This aligns with MICCAI 2026's theme 'From Algorithms to Clinical Translation' by making individual medical AI algorithms composable and deployable through a unified agent framework.
